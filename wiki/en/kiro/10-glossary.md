# Kiro Glossary

> Quick reference for terms used across the Kiro article series.

## How to use this glossary

This is a reference document, not reading material. Look up a specific term when you encounter it and want a plain-language definition. Each term links to the article where it is introduced in context.

---

## Agentic Chat

**Definition**: The conversational AI interface inside Kiro's IDE. Unlike standard chat tools that only output text, Agentic Chat can read and modify files, run commands, and execute multi-step tasks within the project.

**Also known as**: Kiro chat, AI chat panel

**Used in**: [Kiro Core Concepts: Environment](03a-environment-overview.md)

**Korean**: 에이전틱 챗 (Agentic Chat)
**Japanese**: エージェンティックチャット (Agentic Chat)

---

## CLI (Command-Line Interface)

**Definition**: A text-based interface for Kiro invoked from a terminal window. Provides the same AI capabilities as the IDE without a graphical interface, useful for automation and CI/CD pipelines.

**Also known as**: Kiro CLI, terminal interface

**Used in**: [Kiro Core Concepts: Environment](03a-environment-overview.md)

**Korean**: CLI (명령줄 인터페이스)
**Japanese**: CLI（コマンドラインインターフェース）

---

## design.md

**Definition**: The technical design document in a Kiro spec. Describes how a feature will be built: components, data structures, and architecture. Generated by Kiro and reviewed by the developer before implementation begins.

**Also known as**: design document, technical design

**Used in**: [How Kiro Specs Work](04-specs-workflow.md)

**Korean**: 설계 문서 (design.md)
**Japanese**: 設計ドキュメント (design.md)

---

## Hooks

**Definition**: Automated AI actions triggered by file system events. Defined in `.kiro/hooks/` and configured to run a specific AI action when a file matching a pattern is saved, created, or deleted.

**Also known as**: Kiro hooks, file hooks, event hooks

**Used in**: [Kiro Hooks Guide](06-hooks.md), [Kiro Core Concepts: Workflow](03-core-concepts.md)

**Korean**: 훅
**Japanese**: フック

---

## IDE (Integrated Development Environment)

**Definition**: Kiro's graphical interface for writing and running code. Built on Visual Studio Code, it includes a file browser, code editor, Agentic Chat panel, and spec document viewer.

**Also known as**: Kiro IDE, Kiro editor

**Used in**: [What Is Kiro?](01-what-is-kiro.md)

**Korean**: IDE (통합 개발 환경)
**Japanese**: IDE（統合開発環境）

---

## Kiro

**Definition**: An AWS product — an agentic IDE and CLI — that guides AI coding assistants using structured specifications (specs) instead of informal chat prompts. Available at kiro.dev. As of 2026-05-19.

**Also known as**: AWS Kiro, Kiro IDE

**Used in**: [What Is Kiro?](01-what-is-kiro.md)

**Korean**: Kiro (키로)
**Japanese**: Kiro (キロ)

---

## MCP (Model Context Protocol)

**Definition**: An open standard that lets Kiro connect to external tools, databases, APIs, and documentation sources. MCP allows the AI to access information outside the project directory, such as a live database schema or internal API documentation.

**Also known as**: Model Context Protocol

**Used in**: [Kiro MCP Guide](07-mcp.md), [Kiro Core Concepts: Environment](03a-environment-overview.md)

**Korean**: MCP (모델 컨텍스트 프로토콜)
**Japanese**: MCP（モデルコンテキストプロトコル）

---

## requirements.md

**Definition**: The requirements document in a Kiro spec. Contains user stories and acceptance criteria that define what the user should be able to do. Generated by Kiro from an initial feature description and reviewed by the developer.

**Also known as**: requirements document, user stories document

**Used in**: [How Kiro Specs Work](04-specs-workflow.md)

**Korean**: 요구사항 문서 (requirements.md)
**Japanese**: 要件ドキュメント (requirements.md)

---

## Spec (Specification)

**Definition**: In Kiro, a spec is a set of three structured documents — requirements.md, design.md, and tasks.md — that together define what a feature is, how it will be built, and what steps to follow. Stored in `.kiro/specs/[feature-name]/`.

**Also known as**: Kiro spec, specification, spec documents

**Used in**: [How Kiro Specs Work](04-specs-workflow.md), [Kiro Core Concepts: Workflow](03-core-concepts.md)

**Korean**: 명세서 (스펙)
**Japanese**: 仕様書（スペック）

---

## Steering

**Definition**: Persistent, project-wide guidance files stored in `.kiro/steering/`. They instruct the AI on conventions, architecture rules, naming standards, and library choices that apply to the entire project — not just one feature.

**Also known as**: steering files, Kiro steering

**Used in**: [Kiro Steering Guide](05-steering.md), [Kiro Core Concepts: Workflow](03-core-concepts.md)

**Korean**: 스티어링 (steering)
**Japanese**: ステアリング

---

## tasks.md

**Definition**: The task list document in a Kiro spec. An ordered checklist of implementation steps derived from design.md. Each task is independently completable and verifiable against the acceptance criteria in requirements.md.

**Also known as**: task list, implementation tasks

**Used in**: [How Kiro Specs Work](04-specs-workflow.md)

**Korean**: 작업 목록 (tasks.md)
**Japanese**: タスクリスト (tasks.md)

---

## Vibe coding

**Definition**: A development approach where someone gives informal, conversational instructions to an AI tool without a formal specification or plan. Fast for simple tasks; produces inconsistent results as project complexity grows. Kiro is designed as the structured alternative.

**Also known as**: vibe-driven development, informal prompting

**Used in**: [What Is Kiro?](01-what-is-kiro.md), [Why Kiro Matters](02-why-kiro-matters.md)

**Korean**: 바이브 코딩 (vibe coding)
**Japanese**: バイブコーディング (vibe coding)
