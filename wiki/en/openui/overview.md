# OpenUI Overview

> Explains the 4 core components of the OpenUI Lang pipeline and how they work together.

**Official Docs**: [https://www.openui.com/docs/openui-lang/overview](https://www.openui.com/docs/openui-lang/overview)

![OpenUI Banner — Generative UI Framework](https://raw.githubusercontent.com/thesysdev/openui/main/assets/banner.png)

---

## Overview

OpenUI transforms text generated by an LLM into structured UI in real time. The core idea is to treat **model output not as plain text, but as a structured UI specification**.

The pipeline consists of 4 stages:

```
Library → Prompt Generator → LLM → Parser → Renderer → Live UI
```

---

## Core Principles

### 1. Library

A set of components defined with Zod schemas and React renderers.

- Each component defines its props via a Zod schema and specifies how to render via a React component.
- The library defines the "contract" for components and serves as the single source of truth for both prompt generation and rendering.
- Built-in libraries: `openuiLibrary` (general-purpose), `openuiChatLibrary` (chat-optimized)

### 2. Prompt Generator

Automatically generates LLM system prompts from the library.

- Calling `library.prompt()` produces a system prompt containing the list of allowed components, props, and OpenUI Lang syntax rules.
- When the component library changes, prompts are updated automatically.
- Supports both static generation (`@openuidev/cli`) and dynamic generation (`generatePrompt`).

### 3. Parser

Parses OpenUI Lang streamed by the LLM in real time.

- **Line-by-line parsing**: Parses each line as tokens arrive, enabling streaming rendering.
- Automatically excludes invalid output and renders only valid parts.
- Simple `identifier = Expression` syntax minimizes token count.

### 4. Renderer

Maps parsed elements to React components and renders the UI.

- The `<Renderer />` React component receives parsed results and builds UI in real time.
- Uses positional arguments to map to components without additional parsing overhead.
- After LLM generation completes, the UI runs independently (data fetching, state management, event handling).

---

## OpenUI Lang Syntax Basics

OpenUI Lang is line-oriented, maximizing token efficiency.

```
# Positional arguments (compact)
stack = Stack([child1, child2], "row", "l")

# Compare with JSON (verbose)
# { "type": "Stack", "direction": "row", "gap": "l", "children": [...] }
```

Positional arguments are automatically mapped following the Zod schema key order.

## Example Usage

```tsx
import { createLibrary, Renderer } from "@openuidev/react-lang";

// 1. Define library
const library = createLibrary({ /* component definitions */ });

// 2. Generate system prompt
const systemPrompt = library.prompt();

// 3. Stream from LLM + render
<Renderer library={library} stream={llmStream} />
```

## Comparison

| Aspect | OpenUI Lang | JSON-based UI generation |
|--------|------------|--------------------------|
| Token efficiency | High (positional args, line-oriented) | Low (repeated key-value pairs) |
| Streaming | Line-by-line immediate rendering | Parse entire response first |
| Type safety | Zod schemas | None |
| Extensibility | Add custom components | Limited |
