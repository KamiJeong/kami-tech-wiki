# SDD (Spec-Driven Development) 개요

Spec-Driven Development (SDD)는 코드를 작성하기 전에 명세(Specification)를 먼저 작성하는 소프트웨어 개발 방법론입니다. 명세가 구현의 단일 진실 공급원(Single Source of Truth)이 되어, 팀 전체가 동일한 기준으로 작업할 수 있도록 합니다.

이 카테고리에서는 SDD의 개념, 핵심 원리, 그리고 실무에서 어떻게 활용되는지를 다룹니다.
