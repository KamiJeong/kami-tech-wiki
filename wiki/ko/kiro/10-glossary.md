# Kiro 용어집

> Kiro 아티클 시리즈 전반에서 사용되는 용어의 빠른 참조.

## 이 용어집 사용 방법

이 문서는 읽기 위한 자료가 아닌 참조 자료입니다. 특정 용어를 만났을 때 평문 정의가 필요하면 찾아보세요. 각 용어는 맥락 속에서 처음 소개된 아티클로 연결됩니다.

---

## 에이전틱 챗 (Agentic Chat)

**정의**: Kiro IDE 내부의 대화형 AI 인터페이스. 텍스트만 출력하는 일반 채팅 도구와 달리, 에이전틱 챗은 파일을 읽고 수정하며, 명령을 실행하고, 프로젝트 내에서 여러 단계의 작업을 수행할 수 있습니다.

**다른 표현**: Kiro 채팅, AI 채팅 패널

**등장 아티클**: [Kiro 핵심 개념: 환경](03a-environment-overview.md)

**한국어**: 에이전틱 챗 (Agentic Chat)
**일본어**: エージェンティックチャット (Agentic Chat)

---

## CLI (명령줄 인터페이스)

**정의**: 터미널 창에서 실행하는 Kiro용 텍스트 기반 인터페이스. 그래픽 인터페이스 없이 IDE와 동일한 AI 기능을 제공하며, 자동화와 CI/CD 파이프라인에 유용합니다.

**다른 표현**: Kiro CLI, 터미널 인터페이스

**등장 아티클**: [Kiro 핵심 개념: 환경](03a-environment-overview.md)

**한국어**: CLI (명령줄 인터페이스)
**일본어**: CLI（コマンドラインインターフェース）

---

## design.md (설계 문서)

**정의**: Kiro 명세서의 기술 설계 문서. 기능이 어떻게 만들어질지를 설명합니다: 컴포넌트, 데이터 구조, 아키텍처. Kiro가 생성하고 구현 시작 전에 개발자가 검토합니다.

**다른 표현**: 설계 문서, 기술 설계

**등장 아티클**: [Kiro 명세서 워크플로](04-specs-workflow.md)

**한국어**: 설계 문서 (design.md)
**일본어**: 設計ドキュメント (design.md)

---

## 훅 (Hooks)

**정의**: 파일 시스템 이벤트에 의해 트리거되는 자동화된 AI 작업. `.kiro/hooks/`에 정의되며, 패턴과 일치하는 파일이 저장, 생성, 삭제될 때 특정 AI 작업을 실행하도록 설정됩니다.

**다른 표현**: Kiro 훅, 파일 훅, 이벤트 훅

**등장 아티클**: [Kiro 훅 가이드](06-hooks.md), [Kiro 핵심 개념: 워크플로](03-core-concepts.md)

**한국어**: 훅
**일본어**: フック

---

## IDE (통합 개발 환경)

**정의**: 코드 작성 및 실행을 위한 Kiro의 그래픽 인터페이스. Visual Studio Code 기반으로 구축되었으며, 파일 탐색기, 코드 에디터, 에이전틱 챗 패널, 명세서 문서 뷰어를 포함합니다.

**다른 표현**: Kiro IDE, Kiro 에디터

**등장 아티클**: [Kiro란 무엇인가?](01-what-is-kiro.md)

**한국어**: IDE (통합 개발 환경)
**일본어**: IDE（統合開発環境）

---

## Kiro

**정의**: AWS 제품 — 에이전틱(agentic) IDE이자 CLI — 로, 비공식 채팅 프롬프트 대신 구조화된 명세서(스펙)를 사용해 AI 코딩 어시스턴트를 안내합니다. kiro.dev에서 이용 가능. 2026-05-19 기준.

**다른 표현**: AWS Kiro, Kiro IDE

**등장 아티클**: [Kiro란 무엇인가?](01-what-is-kiro.md)

**한국어**: Kiro (키로)
**일본어**: Kiro (キロ)

---

## MCP (Model Context Protocol, 모델 컨텍스트 프로토콜)

**정의**: Kiro가 외부 도구, 데이터베이스, API, 문서 소스에 연결할 수 있게 하는 개방형 표준. MCP를 통해 AI는 라이브 데이터베이스 스키마나 내부 API 문서 같이 프로젝트 디렉토리 외부의 정보에 접근할 수 있습니다.

**다른 표현**: 모델 컨텍스트 프로토콜

**등장 아티클**: [Kiro MCP 가이드](07-mcp.md), [Kiro 핵심 개념: 환경](03a-environment-overview.md)

**한국어**: MCP (모델 컨텍스트 프로토콜)
**일본어**: MCP（モデルコンテキストプロトコル）

---

## requirements.md (요구사항 문서)

**정의**: Kiro 명세서의 요구사항 문서. 사용자가 할 수 있어야 하는 것을 정의하는 사용자 스토리와 인수 기준을 담습니다. Kiro가 초기 기능 설명으로부터 생성하고 개발자가 검토합니다.

**다른 표현**: 요구사항 문서, 사용자 스토리 문서

**등장 아티클**: [Kiro 명세서 워크플로](04-specs-workflow.md)

**한국어**: 요구사항 문서 (requirements.md)
**일본어**: 要件ドキュメント (requirements.md)

---

## 명세서 (스펙, Spec)

**정의**: Kiro에서 명세서(스펙)는 requirements.md, design.md, tasks.md 세 가지 구조화된 문서의 모음으로, 기능이 무엇인지, 어떻게 만들어질지, 어떤 단계를 따를지를 함께 정의합니다. `.kiro/specs/[기능명]/`에 저장됩니다.

**다른 표현**: Kiro 명세서, 스펙, 명세서 문서

**등장 아티클**: [Kiro 명세서 워크플로](04-specs-workflow.md), [Kiro 핵심 개념: 워크플로](03-core-concepts.md)

**한국어**: 명세서 (스펙)
**일본어**: 仕様書（スペック）

## 스티어링 (Steering)

**정의**: `.kiro/steering/`에 저장된 지속적이고 프로젝트 전반의 가이드라인 파일. 전체 프로젝트에 적용되는 규칙 — 특정 기능에만 해당하는 것이 아닌 — 을 AI에게 알려줍니다: 규칙, 아키텍처 원칙, 명명 표준, 라이브러리 선택.

**다른 표현**: 스티어링 파일, Kiro 스티어링

**등장 아티클**: [Kiro 스티어링 가이드](05-steering.md), [Kiro 핵심 개념: 워크플로](03-core-concepts.md)

**한국어**: 스티어링 (steering)
**일본어**: ステアリング

---

## tasks.md (작업 목록 문서)

**정의**: Kiro 명세서의 작업 목록 문서. design.md에서 도출된 순서가 있는 구현 단계의 체크리스트. 각 작업은 독립적으로 완료 가능하고 requirements.md의 인수 기준에 대해 검증 가능합니다.

**다른 표현**: 작업 목록, 구현 작업

**등장 아티클**: [Kiro 명세서 워크플로](04-specs-workflow.md)

**한국어**: 작업 목록 (tasks.md)
**일본어**: タスクリスト (tasks.md)

## 바이브 코딩 (Vibe Coding)

**정의**: 공식 명세서나 계획 없이 AI 도구에 비공식적이고 대화 형식의 지시를 내리는 개발 방식. 간단한 작업에는 빠르지만, 프로젝트 복잡성이 커질수록 일관성 없는 결과를 만들어냅니다. Kiro는 구조화된 대안으로 설계되었습니다.

**다른 표현**: 바이브 기반 개발, 비공식 프롬프팅

**등장 아티클**: [Kiro란 무엇인가?](01-what-is-kiro.md), [왜 Kiro가 중요한가?](02-why-kiro-matters.md)

**한국어**: 바이브 코딩 (vibe coding)
**일본어**: バイブコーディング (vibe coding)
