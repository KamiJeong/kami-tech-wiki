# Kiro란 무엇인가?

> Kiro는 AWS가 만든 에이전틱(agentic) IDE 및 CLI로, 비정형 채팅 프롬프트 대신 구조화된 명세서(스펙)를 사용해 AI 코딩 어시스턴트를 안내합니다.

## 이 페이지의 대상 독자

이 페이지는 프로덕트 매니저, 스타트업 창업자, AI 개발 도구에 관심 있는 모든 분을 위한 글입니다. Kiro가 무엇인지, 자신의 업무와 관련이 있는지 파악하고자 하는 분에게 적합합니다. 기술적 배경 지식은 필요하지 않습니다.

## 왜 중요한가

AI 코딩 어시스턴트를 써본 분이라면 누구나 비슷한 경험을 했을 것입니다. AI에게 무언가를 만들어 달라고 했더니 원하던 것과 다른 결과가 나왔거나, 처음에는 괜찮았지만 비슷한 요청을 다시 했을 때 전혀 다른 결과가 나왔던 경험 말이죠. 프로젝트가 복잡해질수록 이 문제는 더 심각해집니다.

Kiro는 이 문제를 해결하기 위해 만들어졌습니다. AI를 대체하는 것이 아니라, AI에게 작업할 구조화된 지침을 제공하여 결과를 더 예측 가능하고 일관성 있게 만드는 것이 목표입니다.

Kiro를 이해하는 것이 중요한 이유는 AI 코딩 도구가 소프트웨어 개발의 표준이 되어가고 있기 때문입니다. 이 도구들을 사용할지 말지의 문제가 아니라, 어떻게 잘 활용할지의 문제입니다. Kiro는 그 질문에 대한 하나의 대답입니다.

## 쉽게 이해하는 설명

Kiro는 Amazon Web Services(AWS)가 만든 소프트웨어 도구로, kiro.dev에서 이용할 수 있습니다. IDE(통합 개발 환경 — 개발자가 코드를 작성하고 실행하는 프로그램)와 CLI(명령줄 인터페이스 — 터미널 창에서 사용하는 텍스트 기반 도구) 두 가지 형태로 제공됩니다.

Kiro의 핵심은 간단한 원칙에 있습니다. AI가 코드를 작성하기 전에, 먼저 명세서(스펙)를 작성합니다. 명세서는 무엇을 만들 것인지, '완료'가 어떤 상태인지를 설명하는 짧고 구조화된 문서입니다. AI는 비정형 대화 프롬프트가 아닌 이 명세서를 기반으로 작업합니다.

이러한 접근 방식을 명세서 중심 개발(SDD, Spec-Driven Development)이라고 합니다. Kiro는 SDD를 쉽고 자동화된 방식으로 실천할 수 있도록 만들어진 IDE입니다.

Kiro는 2026년 5월 19일 기준 프리뷰 및 활발한 개발 단계에 있었습니다. 기능, 가격, 지역별 제공 여부는 이 아티클 작성 이후 변경되었을 수 있습니다. 최신 정보는 공식 사이트 [kiro.dev](https://kiro.dev)를 확인하세요.

## 핵심 개념

### 에이전틱 IDE (Agentic IDE)

에이전틱(agentic) IDE는 AI가 단순히 코드를 제안하는 것을 넘어, 파일을 읽고 코드를 작성하며 명령을 실행하고 여러 단계의 작업을 수행할 수 있는 개발 환경입니다. Kiro의 IDE는 광범위하게 사용되는 코드 편집기인 Visual Studio Code(VS Code)를 기반으로 구축되어, 기존 확장 기능과 설정을 그대로 활용할 수 있습니다.

**왜 중요한가**: 대부분의 AI 코딩 도구는 '제안' 도구입니다. 작성할 코드를 보여주지만 실제로 타이핑은 사용자가 합니다. 에이전틱 도구는 전체 구현 계획을 수행하고 결과를 검토할 수 있게 해줍니다.

**예시**: 로그인 기능에 대한 명세서를 작성하면, Kiro가 파일, 테스트, 데이터베이스 스키마를 포함한 전체 기능을 단 한 줄의 코드도 직접 입력하지 않고 구현합니다.

### CLI (명령줄 인터페이스)

Kiro는 CLI로도 동작합니다. 전체 IDE를 열지 않고 터미널 창에서 호출할 수 있어 자동화된 워크플로우나 기존 개발 파이프라인에 통합할 때 유용합니다.

**왜 중요한가**: 모든 워크플로우가 그래픽 IDE에서 시작되지는 않습니다. CLI는 서버 환경, CI/CD 파이프라인, 터미널 기반 도구를 선호하는 개발자에게 Kiro를 접근 가능하게 합니다.

**예시**: 팀원이 CI 파이프라인에서 `kiro implement --spec feature-001` 명령어를 실행하여 자동으로 구현 작업을 시작합니다.

### 바이브 코딩 (Vibe Coding) — Kiro가 해결하는 문제

바이브 코딩(vibe coding)은 공식적인 계획 없이 AI 도구에 자유로운 대화 형식으로 지시를 내리는 개발 방식입니다. 즉흥적이고 직관적인 특성을 가지며, 간단한 작업에는 빠르지만 프로젝트가 복잡해질수록 일관성 없는 결과를 낳습니다.

**왜 중요한가**: Kiro는 바이브 코딩에 대한 구조적 대안으로 설계되었습니다. 바이브 코딩이 무엇인지 이해하면 Kiro가 해결하려는 문제를 이해할 수 있습니다.

**예시**: 바이브 코딩 — "AI야, 사용자 데이터를 저장하는 버튼을 추가해." Kiro 방식 — 어떤 데이터를 저장할지, 성공/실패 시 어떻게 되는지, 인수 기준이 무엇인지를 명세서로 작성합니다. AI는 그 명세서를 기반으로 구현합니다.

## 예시와 비유

> 인테리어 공사를 의뢰한다고 상상해 보세요. "예쁘게 해주세요"라고 하면 시공업자가 자신이 좋아하는 방향으로 결정합니다. 캐비닛 색상, 레이아웃, 자재가 모두 당신의 취향과 다를 수 있습니다. 결과물은 나오겠지만, 원하던 결과가 아닐 수 있죠.
>
> 이번에는 설계도를 제공한다고 상상해 보세요. 정확한 치수, 지정된 자재, 납품물 목록, 검수 체크리스트. 시공업자는 당신이 실제로 원하는 것을 만들기 위한 모든 정보를 갖게 됩니다.

Kiro는 당신의 막연한 AI 프롬프트를 설계도로 바꿔주는 시스템입니다. 그리고 그 설계도를 AI 에이전트에게 전달합니다.

## 실용적 활용

**시나리오 1 — 새 기능 개발**: 개발자가 기존 애플리케이션에 새 기능을 추가하려 합니다. 대화식으로 AI에게 프롬프트를 보내는 대신, Kiro를 사용해 먼저 명세서를 작성합니다. Kiro의 AI가 명세서를 기반으로 기능을 구현하여 일관되고 검토 가능한 결과를 만들어냅니다.

**시나리오 2 — 팀 일관성**: 세 명의 개발자로 구성된 팀이 공유 스티어링 파일(프로젝트 전체 가이드 문서)을 사용해 Kiro를 활용합니다. Kiro는 매번 실행 시 공유 가이드를 읽으므로, 모든 개발자의 AI 생성 코드가 동일한 규칙을 따릅니다.

**시나리오 3 — 비개발자 참여**: 프로덕트 매니저가 기능이 무엇을 해야 하는지 평범한 언어로 명세서를 작성합니다. 개발자는 그 명세서를 Kiro에서 활용하여 구현을 안내합니다. 명세서가 두 역할 모두의 공통 문서가 됩니다.

## 흔한 오해

1. **"Kiro는 그냥 또 다른 AI 챗봇이다."** — Kiro는 챗봇이 아닙니다. 명세서 중심의 에이전틱 소프트웨어 개발을 위해 설계된 IDE 및 CLI입니다. 대화형 인터페이스는 Kiro 기능의 일부에 불과합니다.
2. **"Kiro가 개발자를 대체한다."** — Kiro는 반복적인 구현 작업을 자동화하여 개발자를 돕습니다. 개발자는 여전히 명세서를 정의하고, 결과물을 검토하며, 무엇을 만들지 결정합니다. Kiro는 실행을 담당합니다.
3. **"Kiro를 사용하려면 개발자여야 한다."** — Kiro를 이해하는 데 기술적 지식은 필요하지 않습니다. 비기술적 기여자(프로덕트 매니저, 디자이너)도 개발자와 Kiro가 활용할 수 있는 평범한 언어로 명세서를 작성할 수 있습니다.

## 심화 내용

> **심화 독자를 위한 내용** — Kiro가 무엇인지 이해하는 데 이 섹션을 읽을 필요는 없습니다.

Kiro의 IDE는 VS Code 확장 API 기반으로 구축되었으며, GitHub Copilot과 유사한 아키텍처 패턴을 공유합니다. CLI 컴포넌트는 IDE와 동일한 기반 AI 모델과 명세서 시스템을 사용하므로, 대부분의 자동화 작업에서 기능적으로 동등합니다. Kiro의 명세서 시스템(requirements.md, design.md, tasks.md)은 공식 시스템 엔지니어링의 세 문서 패턴을 빠르게 움직이는 소프트웨어 팀에 맞게 적용한 것입니다.
