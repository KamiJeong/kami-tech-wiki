# Kiro MCP 가이드

> MCP(Model Context Protocol, 모델 컨텍스트 프로토콜)는 Kiro가 외부 도구, 데이터베이스, API, 문서 소스에 연결할 수 있게 하는 개방형 표준으로, AI가 프로젝트 파일 외부에서도 정보를 보고 작업할 수 있도록 합니다.

## 이 페이지의 대상 독자

이 페이지는 Kiro에 외부 시스템 — 데이터베이스, 내부 API, 서드파티 문서, 프로젝트 관리 도구 — 에 대한 접근 권한을 주고 싶은 개발자를 위한 글입니다. API에 대한 기본적인 이해가 있으면 도움이 되지만 필수는 아닙니다.

## 왜 이 내용이 중요한가

기본적으로 Kiro의 AI 에이전트는 프로젝트 디렉토리 내부의 파일만 볼 수 있습니다. 많은 작업에는 이것으로 충분합니다. 그러나 외부 시스템에 의존하는 프로젝트 — 라이브 데이터베이스, 특정 응답 형식을 가진 서드파티 API, 회사 내부 문서 — 에서 로컬 파일만으로 작업한다는 것은 AI가 실제로 알 수 있는 것들을 추측해야 한다는 의미입니다.

MCP가 그 격차를 해소합니다. MCP 연결을 설정함으로써, 팀은 Kiro에 외부 맥락에 대한 접근을 제공합니다: 실제 데이터베이스 스키마, 진짜 API 응답 형식, 내부 문서 사이트. AI가 추측 대신 실제 데이터로 작업하기 때문에 결과물이 더 정확해집니다.

## 쉬운 설명

MCP는 Model Context Protocol(모델 컨텍스트 프로토콜)의 약자입니다. Kiro 전용 기술이 아닌 개방형 표준으로, AI 모델이 외부 도구와 데이터 소스로부터 정보를 요청하고 받는 공통 방법을 정의합니다.

MCP를 AI 도구를 위한 범용 플러그 표준으로 생각해보세요. USB 포트가 USB 호환 기기를 컴퓨터에 연결할 수 있게 하듯, MCP는 Kiro가 MCP 호환 데이터 소스나 도구에 연결할 수 있게 합니다. 더 많은 도구들이 이 표준을 채택할수록 사용 가능한 MCP 커넥터도 늘어납니다.

Kiro에서 MCP를 사용하려면 Kiro 설정에서 어떤 외부 소스에 연결할지를 설정합니다. AI가 외부 소스의 정보가 필요할 때 — 예를 들어 데이터베이스 테이블의 현재 구조 — MCP 연결을 통해 요청을 보내고 정보를 받습니다.

이 과정은 투명하게 진행됩니다. 데이터베이스 스키마를 파일에 직접 복사해 프롬프트에 붙여넣을 필요가 없습니다. Kiro의 AI가 MCP 연결을 통해 자동으로 정보를 가져옵니다.

## 핵심 개념

### 외부 데이터 소스

MCP는 Kiro가 데이터 소스에 연결할 수 있게 합니다: 데이터베이스, 프로젝트 디렉토리 외부의 파일 시스템, 문서 서버, 그리고 정보를 제공하는 다른 시스템들.

**왜 중요한가**: 프로젝트가 데이터베이스에 의존할 때, AI는 정확한 쿼리나 ORM 코드를 생성하기 위해 테이블 구조를 알아야 합니다. MCP를 통해 Kiro는 실제 데이터베이스 스키마를 읽을 수 있습니다. MCP 없이는 스키마를 수동으로 알려주거나 맥락으로부터 추측해야 합니다.

**예시**: PostgreSQL 데이터베이스에 MCP 연결이 설정된 Kiro는 `users` 테이블에 `id (uuid)`, `email (varchar)`, `created_at (timestamp)` 컬럼이 있다는 것을 읽을 수 있습니다. 사용자 쿼리 함수를 생성할 때, Kiro는 스키마를 복사해 붙여넣을 필요 없이 올바른 컬럼 이름과 타입을 사용합니다.

### 외부 도구와 서비스

MCP는 Kiro가 도구 — 입력을 받아 작업을 수행하는 시스템 — 에도 연결할 수 있게 합니다. 웹 검색, 이슈 트래커, CI/CD 시스템, 코드 리뷰 도구 등이 그 예입니다.

**왜 중요한가**: Kiro가 외부 도구와 상호작용할 수 있을 때, 코드 생성 이상의 것을 할 수 있습니다. GitHub 이슈의 현재 상태를 가져오거나, 의존성의 최신 버전을 조회하거나, 테스트 스위트가 통과했는지 확인할 수 있습니다.

**예시**: GitHub 이슈 트래커에 대한 MCP 연결은 개발자가 수락 기준을 명세서에 직접 복사하는 대신, Kiro가 연결된 이슈에서 직접 수락 기준을 읽을 수 있게 합니다.

### 개방형 표준 생태계

MCP는 개방형 표준으로 유지됩니다. 이는 서드파티 개발자들이 어떤 서비스를 위한 MCP 커넥터도 만들 수 있고, 그 커넥터들이 Kiro뿐만 아니라 모든 MCP 호환 AI 도구에서 작동한다는 것을 의미합니다.

**왜 중요한가**: MCP를 채택한다는 것은 Kiro가 사전 구축된 커넥터들의 성장하는 생태계로부터 혜택을 받는다는 것을 의미합니다. 한 MCP 호환 도구를 위해 만들어진 커넥터는 Kiro 전용 통합이 필요하지 않아도 Kiro에 적용될 수 있습니다.

**예시**: 문서 플랫폼을 위해 만들어진 MCP 커넥터를 Kiro에서 설정하면, Kiro 전용 통합 없이도 AI가 해당 플랫폼의 내용에 접근할 수 있게 됩니다.

## 비유와 예시

> 책상 위의 문서만 읽을 수 있는 조사 어시스턴트는 유용하지만 제한적입니다. 도서관에 전화하고, 회사 데이터베이스를 확인하고, 최신 뉴스를 조회할 수 있는 조사 어시스턴트는 훨씬 더 유용합니다 — 더 똑똑해서가 아니라 더 많은 정보에 접근할 수 있기 때문입니다.

MCP는 Kiro에게 전화와 도서관 카드를 주는 것과 같습니다. AI 자체는 같습니다. 달라지는 것은 결과물을 생성하기 전에 접근할 수 있는 정보의 범위입니다.

## 실제 활용 사례

**시나리오 1 — 데이터베이스를 인식하는 코드 생성**: Kiro를 MCP를 통해 개발 데이터베이스에 연결합니다. 데이터 접근 코드를 생성할 때 Kiro는 실제 테이블 스키마를 읽어 실제 구조와 일치하는 쿼리를 생성합니다. 수동 스키마 문서화가 필요 없습니다.

**시나리오 2 — 내부 문서 조회**: Kiro를 회사의 내부 문서 사이트에 연결합니다. 내부 API를 호출하는 코드를 생성할 때 Kiro가 실제 API 문서를 참조해 엔드포인트 경로, 매개변수, 응답 형식을 정확하게 처리합니다.

**시나리오 3 — 이슈 트래커 통합**: Kiro를 GitHub Issues에 연결합니다. 새 기능 명세서를 시작할 때, Kiro가 처음부터 시작하지 않고 관련 이슈에서 맥락 — 사용자 스토리, 수락 기준, 기존 논의 — 을 읽습니다.

**기본 MCP 연결 설정**:
1. Kiro의 설정 패널을 엽니다.
2. MCP 연결 섹션으로 이동합니다.
3. 연결 추가: 커넥터 유형(데이터베이스, 웹 검색, 커스텀)을 선택하고 연결 정보를 입력합니다.
4. 연결을 테스트해 Kiro가 외부 소스에 접근할 수 있는지 확인합니다.
5. 연결 사용: 에이전틱 챗 또는 명세서 생성에서 Kiro가 관련 시 연결된 소스를 자동으로 활용합니다.

## 흔한 오해

1. **"MCP는 기본적으로 Kiro에게 인터넷 접근을 준다."** — MCP 연결은 명시적으로 설정됩니다. Kiro는 자동으로 인터넷을 탐색하지 않습니다. 각 연결은 설정되고 승인되어야 합니다.
2. **"MCP는 Kiro 전용 기능이다."** — MCP는 개방형 표준입니다. 여러 AI 도구와 함께 작동하도록 설계되었습니다. Kiro가 MCP를 지원하지만, MCP는 Kiro 전용이 아닙니다.
3. **"MCP 설정에는 개발자가 필요하다."** — 기본 MCP 연결(웹 검색, 로컬 파일 경로)은 코드 작성 없이 Kiro의 설정 UI를 통해 설정할 수 있는 경우가 많습니다. 복잡한 커스텀 통합은 개발자 참여가 필요할 수 있습니다.

## 심화 내용

> **심화 독자를 위한 내용** — 이 섹션을 읽지 않아도 Kiro의 MCP 통합을 이해하는 데 지장이 없습니다.

MCP는 클라이언트-서버 모델을 사용합니다. Kiro는 MCP 클라이언트로 작동하고, 외부 데이터 소스들은 MCP 서버를 노출합니다. 프로토콜은 클라이언트가 정보(리소스)를 요청하고, 작업(도구)을 호출하고, 구조화된 응답을 받는 방법을 정의합니다. 커스텀 MCP 서버는 프로그래밍 가능한 인터페이스를 노출하는 모든 시스템 — REST API, GraphQL, SQL 데이터베이스, 파일 시스템, 독점 형식 — 을 위해 구축될 수 있어, 팀이 Kiro의 컨텍스트를 모든 내부 시스템으로 확장할 수 있습니다.
