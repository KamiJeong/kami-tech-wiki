# Kiro

> **最新情報に関する注意**: このページの情報は 2026-05-18 時点のものです。Kiro は現在活発に開発中であり、機能やインターフェースが変更される可能性があります。最新情報は公式サイト [https://kiro.dev](https://kiro.dev) をご確認ください。

## Kiro とは？

Kiro は AWS が開発した AI 搭載統合開発環境 (IDE) です。VS Code を基盤として構築されており、既存の VS Code 拡張機能や設定をそのまま活用できます。加えて、スペック駆動開発 ([[Spec-Driven Development (SDD)|SDD]]) のための専用ワークフローが組み込まれています。

Kiro の核心的な哲学は「仕様がコードより先」です。開発者が自然言語で機能を説明すると、Kiro の AI エージェントが構造化されたスペックドキュメントを自動生成し、それを実装タスクに分解し、コードを記述します。これらすべての作業が仕様を起点として行われます。

## コア概念

### スティアリングファイル (Steering Files)

スティアリングファイルは、Kiro エージェントの動作方法を定義するマークダウンファイルです。プロジェクトのコーディング規約、アーキテクチャの原則、命名規則、禁止パターンを記述し、`.kiro/steering/` ディレクトリ以下に配置します。エージェントはすべてのセッションの開始時にこれらのファイルを参照し、プロジェクト全体を通じて一貫したコード生成を保証します。

### スペックドキュメント (Spec Documents)

Kiro のスペックシステムは3つのファイルで構成されます。

- **requirements.md**: ユーザーストーリーと機能要件を定義します。「ユーザーとして、私は〜できる必要がある」という形式の受け入れ基準が含まれます。
- **design.md**: 技術的アーキテクチャ、データモデル、API コントラクト、コンポーネント構造を定義します。コードが一行も書かれる前に「どのように作るか」を明確にします。
- **tasks.md**: 設計から導き出された依存関係順に整理された実装タスクのリストです。各タスクは独立して完了可能であり、要件へのトレーサビリティを持ちます。

### エージェントタスク (Agent Tasks)

エージェントタスクは、仕様から導き出された具体的な実装単位です。開発者はタスクを選択して Kiro エージェントに割り当てます。エージェントは関連するスペックドキュメントとスティアリングファイルを参照してタスクを実装し、受け入れ基準に対する自己チェックを実行してからタスクを完了とします。

### オートパイロットモード (Autopilot Mode)

オートパイロットモードは、Kiro が人の介入なしに仕様に定義されたすべてのタスクを順次完了する自律実行モードです。開発者は仕様を定義してオートパイロットを有効化し、エージェントが全実装を完了した後に結果をレビューします。詳細な仕様が作成された機能のスキャフォールディングや、反復的な実装作業に特に有効です。

## 使用シナリオ

| 適切な場合 | 適切でない場合 |
|----------|--------------|
| 仕様を先に定義してAIに実装させる場合 | 既存の完成したコードを単純に編集する場合 |
| チーム全体で一貫したコーディング規約を維持する場合 | 事前設計なしに素早くプロトタイプを作成する場合 |
| 複雑な機能を追跡可能なタスクに分解して段階的に実装する場合 | Kiro がサポートしていない言語やフレームワークを使用する場合 |

## クイックスタート

Kiro で新機能を開発する基本的な流れは次のとおりです。

```bash
# 1. Kiro をインストールしてプロジェクトを開く
#    (kiro.dev からインストーラーをダウンロード)

# 2. 新しいスペックを作成する
#    コマンドパレット → "Kiro: New Spec"
#    または .kiro/specs/ 以下に新しいフォルダーを手動作成

# 3. requirements.md に要件を記述する
# 例:
# ## User Stories
# - As a user, I want to log in with email and password
#   - Acceptance: Login completes within 2 seconds
#   - Acceptance: Invalid credentials show an error message

# 4. Kiro に design.md の生成を依頼する
#    "requirements.md に基づいて技術設計を生成してください"

# 5. Kiro に tasks.md の生成を依頼する
#    "設計を実装タスクに分解してください"

# 6. タスクを実行する
#    タスクを選択 → "Run with Kiro Agent"
#    またはオートパイロットを有効化:
#    コマンドパレット → "Kiro: Run Autopilot"
```

**結果**: Kiro エージェントがスペックドキュメントを参照してコードを生成し、完了後に受け入れ基準に基づく自己検証を実行します。

> [!tip] はじめる前に
> 最初のスペックを作成する前にスティアリングファイルを作成しましょう。技術スタック、命名規則、アーキテクチャの原則を `.kiro/steering/conventions.md` に定義することで、以降のすべてのエージェント作業に一貫性が生まれます。

## SDD との関係

Kiro は Spec-Driven Development ([[Spec-Driven Development (SDD)|SDD]]) の IDE レベルの実装です。SDD の「仕様優先 (Spec-First)」の原則は、Kiro の3ドキュメント仕様システムに直接対応します。「生きた仕様 (Living Specification)」の原則は、Kiro が実装の進行に合わせてスペックドキュメントを更新する機能として実装されています。「反復的精練 (Iterative Refinement)」は、タスク単位の実行と検証サイクルとして実装されています。

Kiro を使用することは、実際にはIDEレベルのツールサポートでSDD方法論を実践することです。

## 注意事項 & 制限

> [!warning] 既知の制約
> - **プラットフォームサポート**: 2026-05-18 時点では、Kiro は macOS と Windows をサポートしています。Linux のサポートは限定的な場合があります。最新のサポート状況は公式サイトをご確認ください。
> - **言語・フレームワーク**: すべての言語とフレームワークが同じレベルでサポートされているわけではありません。オートパイロットの精度は仕様の詳細度に大きく依存します。

> [!note] 活発な開発中
> Kiro は 2026 年現在、活発に開発中の製品です。機能、UI、API はアップデートにより変更される可能性があります。このページの情報は 2026-05-18 時点のものです。最新情報は公式ドキュメントをご参照ください。
